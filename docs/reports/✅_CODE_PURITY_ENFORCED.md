## 📋 NEW RULES (5 Critical Points)

### Rule 1: CODE PURITY
```
Code block must be 100% valid programming syntax
```
- No mixed languages
- No explanatory text inside code
- Pure programming code only

### Rule 2: NO KHMER TEXT IN CODE
```
NO explanatory text inside code block
(only standard // or /* */ comments if needed)
```
- Khmer explanations OUTSIDE code blocks only
- Code blocks contain ONLY programming syntax
- Comments use standard format (// or /* */)

### Rule 3: LOGIC FIX
```
Fix all errors (e.g., print() → console.log() in JavaScript)
```
- Correct syntax errors
- Fix wrong function calls
- Ensure code actually works

### Rule 4: COMPLETE CODE
```
Show FULL working code, not snippets
```
- Complete functions/classes
- All necessary imports
- Ready to run

### Rule 5: FALLBACK
```
If code is perfect, add comment:
// កូដនេះត្រឹមត្រូវ និងមានប្រសិទ្ធភាពហើយ (Khmer)
// This code is correct and efficient (English)
```
